# Cover Letter Style

Use this skill when the user provides:

* A job posting
* Resume bullets, work experience, project history, or `resume.md`

## Goal

Write one concise cover letter paragraph that connects the user's real experience to the role.

## Tone

* Professional, confident, and natural.
* Senior but not arrogant.
* Business-aware, not just technical.
* Clear and direct.
* Avoid hype, flattery, and generic passion statements.

## Structure

Write 4–6 sentences in this order:

1. Explain why the user's background fits the role.
2. Match 2–3 job requirements with relevant experience.
3. Include measurable results whenever possible.
4. End with how the user can contribute to the company.

## Keyword Matching

* Extract the most important job requirements.
* Prioritize required skills over nice-to-have skills.
* Match keywords only to real experience.
* Reuse keywords naturally.
* Do not force every keyword into the paragraph.
* Never invent skills, tools, industries, or achievements.
* Rewrite resume bullets into business-focused achievements.

## Resume Mapping

Use these mappings when selecting evidence:

* React/TypeScript booking dashboard → React, TypeScript, frontend development, customer-facing dashboards, UI development, support workflow improvement.
* Node.js/Express REST APIs → backend development, REST API design, Express, validation, error handling, API endpoints, full-stack development.
* PostgreSQL migration → PostgreSQL, MySQL migration, zero-downtime migration, database reliability.
* GitHub Actions CI → CI/CD, GitHub Actions, automated testing, engineering workflow improvement.
* Mentoring interns → code review, mentoring, debugging, collaboration.
* Vue inventory app → Vue.js, internal tools, inventory management, maintenance, feature development.
* Jest testing → Jest, unit testing, test coverage, code quality.
* Stripe integration → Stripe, payment APIs, webhooks, third-party API integration.
* Swagger/OpenAPI → API documentation, developer onboarding.
* Habit Tracker → JavaScript, CSS, dashboard UI, rapid prototyping.
* Repo Health CLI → Node.js, GitHub API, developer tooling, repository analysis.
* Prefer measurable achievements such as:

  * Reduced support-ticket resolution time by 30%.
  * Built 12 REST API endpoints.
  * Migrated MySQL to PostgreSQL with zero downtime.
  * Increased test coverage from 12% to 68%.
  * Prevented 4–5 issues per week through CI.
  * Supported an application used by ~40 internal staff.

## Style Rules

* Write in first person.
* Keep everything in one paragraph unless requested otherwise.
* Prefer evidence over claims.
* Prefer outcomes over responsibilities.
* Keep English simple and natural.
* Avoid phrases like:

  * "I am the perfect candidate."
  * "I am deeply passionate."
  * "I would be honored to apply."

## Output Pattern

Follow this pattern:

> My experience building [project/system] aligns well with your need for [requirement]. I have worked on [relevant experience], including [project] that achieved [business result]. I also bring experience in [second requirement], allowing me to contribute to [team/company goal].
