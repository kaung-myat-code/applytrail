# Cover Letter Style



Use when the user provides resume bullets, work experience, or project history plus a job posting.



## Goal



Draft one concise cover letter paragraph that connects the user's real experience to the role.



## Tone



* Professional, confident, and natural.

* Senior but not arrogant.

* Business-aware, not just technical.

* Clear and direct.

* Suitable for Myanmar/Southeast Asia job applications.

* Avoid hype, flattery, and generic passion statements.



## Structure



Write 4–6 sentences in this order:



1. Start with role fit.

2. Match 2–3 job requirements to the user's experience.

3. Include measurable impact when available.

4. End with how the user can contribute to the company/team.



## Keyword Matching Rules



* Extract the most important keywords from the job posting.

* Prioritize role requirements over nice-to-have skills.

* Match keywords only to experience the user actually has.

* Reuse job-posting keywords naturally, not mechanically.

* Do not force every keyword into the paragraph.

* Do not invent tools, skills, industries, or achievements.

* Translate resume bullets into impact-focused language.



## Preferred Matches



* React/TypeScript booking dashboard → React, TypeScript, frontend development, customer-facing dashboard, UI development, order history, support workflow improvement.

* Node.js/Express REST APIs → backend development, REST API design, Express, input validation, error handling, API endpoints, full-stack development.

* PostgreSQL migration → PostgreSQL, MySQL migration, database migration, zero-downtime migration, rollback planning, data storage, database reliability.

* GitHub Actions CI pipeline → CI/CD, GitHub Actions, automated testing, linting, pull request quality, engineering workflow improvement.

* Mentoring interns → code review, junior developer mentoring, debugging support, collaboration, team contribution.

* Vue inventory app → Vue.js, internal tools, inventory management, bug fixing, feature maintenance, staff-facing applications.

* Jest billing module tests → Jest, unit testing, test coverage improvement, billing systems, code quality.

* Stripe payment integration → payment API integration, Stripe, webhooks, payment confirmation, refunds, third-party API integration.

* Swagger/OpenAPI documentation → API documentation, Swagger, OpenAPI, developer onboarding, internal documentation.

* Habit Tracker project → vanilla JavaScript, CSS, dashboard UI, data persistence, personal project, fast prototyping.

* Repo Health CLI → Node.js CLI, GitHub API, developer tooling, repository analysis, stale branch detection, secret scanning.

* Measurable results → cut support-ticket resolution time by 30%, built 12 REST API endpoints, migrated MySQL to PostgreSQL with zero downtime, caught 4–5 issues per week before merge, raised test coverage from 12% to 68%, supported an app used by ~40 internal staff.



## Style Rules



* Use first person.

* Keep one paragraph unless asked otherwise.

* Prefer evidence over claims.

* Prefer outcomes over duties.

* Prefer specific experience over broad statements.

* Keep English simple and ready to send.

* Avoid phrases like “perfect candidate,” “deeply passionate,” and “honored to apply.”



## Example Pattern



My experience building [project/system type] aligns well with your need for [main job requirement]. I have worked on [relevant experience], including [specific project or responsibility] that helped [measurable or business result]. I also bring experience in [second matching requirement], which would help me contribute to [company/team goal]. This background allows me to deliver practical, maintainable solutions that support both users and business needs.
