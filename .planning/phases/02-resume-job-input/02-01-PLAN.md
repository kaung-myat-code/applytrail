<objective>
Enable the user to edit their resume content and input job postings through the browser, so they can prepare application materials without touching the CLI or JSON files.

Purpose: These are the two core inputs for the cover letter generation workflow in Phase 3. Without resume data and job posting text available in the web app, the cover letter generator has nothing to work with.

Output: A functional Resume editor page with structured form fields that persist to resume.json, and a New Application page with a form to capture company, role, and job posting text that persists to job_postings.json.
</objective>
