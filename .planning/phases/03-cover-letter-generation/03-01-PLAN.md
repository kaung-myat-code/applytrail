Output: A working POST /api/generate-cover-letter endpoint that returns a 4-6 sentence cover letter paragraph.

  <action>
    Create the cover letter generation engine as a separate module. This module is designed to be replaceable — the heuristic approach can later be swapped for an LLM API call.

    **Create directory:** `server/lib/` (if it doesn't exist)

    **Create file:** `server/lib/cover-letter.js`

    Export three functions:

    1. `extractKeywords(postingText)` — Extract meaningful keywords from job posting text.
       - Convert to lowercase, split on non-alphanumeric characters
       - Filter out common stop words (the, a, an, and, or, is, are, with, for, to, in, of, etc.)
       - Remove very short tokens (< 2 chars) and very long tokens (> 30 chars)
       - Return deduplicated array of keyword strings

    2. `matchResumeToJob(resume, postingText)` — Match resume content to job posting keywords.
       - Extract keywords from postingText using extractKeywords()
       - Match against resume.skills (case-insensitive substring matching)
       - Match against resume.experience bullets (check if any keyword appears in bullet text)
       - Match against resume.projects bullets (same approach)
       - Match against resume.summary (check for keyword overlap)
       - Return object: { matchedSkills: string[], matchedExperience: string[], matchedProjects: string[] }
       - Each matchedExperience/matchedProjects entry is the original bullet text

    3. `generateCoverLetter(resume, jobPosting)` — Assemble a cover letter paragraph.
       - Call matchResumeToJob() to get matches
       - Build 4-6 sentences following this structure:
         - Sentence 1: Intro — mention role title and company name, connect to background
         - Sentences 2-3: Match 2-3 job requirements with relevant experience bullets
         - Sentence 4: Include a measurable achievement (look for numbers/percentages in matched bullets)
         - Sentence 5-6: Close — how user can contribute to company
       - Write in first person, professional but natural tone
       - Avoid hype phrases ("I am deeply passionate", "I would be honored")
       - If no matches found, return a generic paragraph noting the user's general background
       - Return the paragraph as a single string

    **Style reference:** Use the tone and structure from `.claude/skills/custom-cover-letter/SKILL.md` as a guide, but implement as code heuristics rather than LLM prompting.

    **Key design decisions:**
    - Module exports functions only — no Express routes (those go in server/index.js)
    - Keyword matching is case-insensitive
    - Prefer measurable achievements (bullets with numbers) over generic bullets
    - Deduplicate matched bullets — don't repeat the same experience twice
    - Keep the paragraph to one block of text (no line breaks within)
  </action>


    **Add route after existing routes (before the PORT listener):**

    ```javascript
    app.post('/api/generate-cover-letter', (req, res) => {
      const { job_posting_id } = req.body

      if (!job_posting_id) {
        return res.status(400).json({ error: 'job_posting_id is required' })
      }

      const postings = readJSON('job_postings.json')
      const posting = postings.find(p => p.id === job_posting_id)

      if (!posting) {
        return res.status(404).json({ error: 'Job posting not found' })
      }

      const resume = readJSON('resume.json')
      const paragraph = generateCoverLetter(resume, posting)

      res.json({ ok: true, cover_letter_paragraph: paragraph })
    })
    ```

    **Route behavior:**
    - Accepts POST with JSON body { job_posting_id }
    - Looks up the job posting by ID from job_postings.json
    - Reads resume.json
    - Calls generateCoverLetter(resume, posting) from the cover-letter module
    - Returns { ok: true, cover_letter_paragraph: "..." }
    - Returns 400 if job_posting_id is missing
    - Returns 404 if posting ID not found in job_postings.json
  </action>
