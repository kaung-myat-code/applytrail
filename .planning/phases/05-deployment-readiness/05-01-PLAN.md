  <action>
    Step 1: Install production middleware packages in the server directory.
    Run `cd /Users/kmpg/VibeCodeTours/applytrail/server && npm install helmet compression`.
    This adds helmet and compression to server/package.json dependencies.

    Step 2: Modify server/index.js with the following changes (in this order within the file):

    a) Add require statements at the top (after existing requires):
       - `const helmet = require('helmet')`
       - `const compression = require('compression')`

    b) Add a health endpoint BEFORE any other routes (per DEPLOY-02):
       `app.get('/api/health', (req, res) => { res.json({ status: 'ok', uptime: process.uptime() }) })`

    c) Add NODE_ENV-based middleware AFTER `app.use(express.json())` but BEFORE API routes (per DEPLOY-03, DEPLOY-04, DEPLOY-05):
       - `if (process.env.NODE_ENV === 'production') { app.use(helmet({ contentSecurityPolicy: false })); app.use(compression()); }`
       - helmet() sets X-Content-Type-Options, X-Frame-Options, etc. CSP is disabled because Vite-built React apps may use inline styles that would be blocked by default CSP.
       - compression() with default settings compresses responses where the client supports it

    d) Add static file serving and SPA catch-all AFTER all existing API routes but BEFORE `app.listen` (per DEPLOY-01, DEPLOY-05):
       - `if (process.env.NODE_ENV === 'production') { ... }` block containing:
       - `app.use(express.static(path.join(__dirname, '..', 'client', 'dist')))`
       - `app.get('*', (req, res) => { res.sendFile(path.join(__dirname, '..', 'client', 'dist', 'index.html')) })`
       - The SPA catch-all uses `*` wildcard which matches all routes — this is why it MUST come after all /api/* routes
       - path.join(__dirname, '..', 'client', 'dist') resolves from server/ up to project root, then into client/dist/

    Step 3: Verify DATA_DIR remains unchanged — `const DATA_DIR = path.join(__dirname, '..')` still points to project root. In production on Render, the working directory is the project root, and __dirname for server/index.js is the server/ directory, so `..` correctly resolves to the project root where JSON data files live.

    Important ordering in server/index.js after changes:
    1. Requires (express, fs, path, cover-letter, helmet, compression)
    2. App creation and DATA_DIR
    3. app.use(express.json())
    4. NODE_ENV production middleware (helmet, compression)
    5. Helper functions (generateId, readJSON, writeJSON, migrateApplications)
    6. Health endpoint (GET /api/health)
    7. All existing API routes (applications, job-postings, resume, generate-cover-letter)
    8. NODE_ENV production static serving + SPA catch-all
    9. PORT and app.listen

    ```yaml

      - type: web
        name: applytrail
        runtime: node
        plan: free
        buildCommand: npm install && npm run build
        startCommand: npm start
        envVars:
          - key: NODE_ENV
            value: production

    ```
