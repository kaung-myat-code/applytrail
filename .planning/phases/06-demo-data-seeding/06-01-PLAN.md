  <action>
Add a `seedDemoData()` function to `server/index.js` that seeds demo data on startup when data files are empty or missing. This implements DEMO-04.

**Function logic:**
1. Define `DEMO_DIR = path.join(__dirname, 'demo-data')` alongside the existing `DATA_DIR`
2. For each of the three data files (`resume.json`, `job_postings.json`, `applications.json`):
   - Check if the file exists in DATA_DIR using `fs.existsSync()`
   - If it does not exist, OR if it exists but is empty (`fs.readFileSync` returns empty string or whitespace-only), OR if it exists but contains only `[]` (empty JSON array) or `{}` (empty JSON object for resume.json):
     - Read the corresponding file from DEMO_DIR
     - Write it to DATA_DIR using `writeJSON()`
     - Log: `Seeded {filename} from demo data`
3. If the file already has content, skip it silently

**Important edge cases for the emptiness check:**
- `resume.json` is an object, not an array. Check for missing file, empty content, or `{}`
- `job_postings.json` and `applications.json` are arrays. Check for missing file, empty content, or `[]`
- Read the raw file content as a string and `JSON.parse()` it, then check if it's an empty array/object. Wrap in try/catch — if JSON is invalid, treat as empty and seed.

**Placement in server/index.js:**
- Add the function definition after the existing `writeJSON()` function (around line 34)
- Call `seedDemoData()` immediately before the existing `migrateApplications()` call (around line 62). Seeding must happen before migration so the demo application records get migrated if needed.

**Do NOT change:**
- The existing `readJSON()`, `writeJSON()`, `generateId()`, or `migrateApplications()` functions
- Any API routes
- The DATA_DIR constant
- The server listen logic
