**Step 2: Create server/lib/analysis/providers/heuristic.js (heuristic provider)**

This module exports a single `analyzeResume(resume, jobPosting)` function that returns a MatchReport object. Per ANALYSIS-01, ANALYSIS-02, ANALYSIS-03.

The MatchReport shape (from research):
```
{
  score: number (0-100),
  summary: string,
  strengths: string[],
  gaps: string[],
  keywords: { matched: string[], missing: string[], bonus: string[] },
  sections: {
    summary: { matchRate: number, matchedItems: string[], missingItems: string[], summary: string },
    skills: { ... same shape ... },
    experience: { ... same shape ... },
    projects: { ... same shape ... },
    education: { ... same shape ... }
  }
}
```

Implementation:
- Import `extractKeywords` and `extractResumeKeywords` from `../keywords.js`
- Extract posting keywords via `extractKeywords(jobPosting.posting_text || '')`
- Extract resume keywords via `extractResumeKeywords(resume)`
- Categorize keywords: matched (posting keyword found in resume keywords via substring match), missing (not found), bonus (resume keyword not in posting keywords)
- Compute score from three weighted components:
  - Keyword coverage (50%): `matched.length / postingKeywords.length * 100` (or 0 if no posting keywords)
  - Section breadth (30%): count of sections with at least one match / 5 * 100
  - Match depth (25%): `Math.min(100, uniqueMatchedItems / 5 * 100)`
  - Final: `Math.round(keywordScore * 0.50 + sectionScore * 0.30 + depthScore * 0.25)`
- Generate section findings for each of the 5 sections (summary, skills, experience, projects, education):
  - Summary: extract keywords from `resume.summary`, compare to posting keywords
  - Skills: compare `resume.skills` array to posting keywords (substring match, case-insensitive)
  - Experience: check each bullet in `resume.experience` for posting keyword presence
  - Projects: check each bullet in `resume.projects` for posting keyword presence
  - Education: check degree, institution, bullets for posting keywords
  - Each section: compute matchRate (0-1), list matchedItems, list missingItems, generate a one-sentence summary
- Generate strengths (sections with matchRate >= 0.5, keyword coverage >= 50%)
- Generate gaps (missing keywords, sections with matchRate < 0.3)
- Generate summary sentence combining score, top strengths, and top gaps

Handle edge cases:
- Empty or missing `posting_text`: return score 0 with message "Add a job posting with description text to run analysis"
- Empty resume (no skills, no experience, no projects): return score 0 with message "Add content to your resume before running analysis"
- Missing fields: use `(resume.skills || [])`, `(resume.experience || [])`, etc.
