# Plan 12-01: Backend â€” Draft Storage API + Patch Application Engine

## Goal

Build the server-side infrastructure for tailored resume generation: a patch application engine that applies accepted suggestions to a deep copy of the source resume, and ephemeral draft storage so accept/reject decisions survive navigation between the review and preview pages.

## Requirements

TAILOR-01, TAILOR-02, TAILOR-04, TAILOR-06

## Tasks

### Task 1: Create applyPatches module

**read_first:**
- `server/lib/analysis/providers/heuristic.js` (suggestion shape: id, section, type, current, suggested, reason)
- `server/lib/analysis/validate.js` (validateMatchReport, validateSuggestions â€” validation patterns)
- `server/index.js` (validateResume function at lines 44-144 â€” the canonical resume schema validator)
- `resume_library/mr3ldtxymun8qiljd.json` (actual resume data shape to test against)

**acceptance_criteria:**
- File `server/lib/tailor/applyPatches.js` exists and exports `applyPatches(resume, suggestions, decisions)`
- `applyPatches` returns `{ resume: deepCopy, validation: { ok: boolean, errors: string[] } }`
- For `summary` section + `modify` type: replaces `resume.summary` with `decisions[id].editedContent || suggestion.suggested`
- For `summary` section + `add` type: sets `resume.summary` to `suggestion.suggested`
- For `skills` section + `add` type: appends `suggestion.suggested` to `resume.skills` array
- For `experience` section + `add` type: appends `suggestion.suggested` as a bullet to the last entry in `resume.experience` (or creates a new entry with `{ company: "", role: "", period: "", bullets: [suggested] }` if experience is empty)
- For `projects` section + `add` type: appends `suggestion.suggested` as a bullet to the last entry in `resume.projects` (or creates a new entry with `{ name: "", description: "", bullets: [suggested] }` if projects is empty)
- `resume.experience[].bullets` entries that match `suggestion.current` are replaced when `type === "modify"` and the suggestion references a specific bullet (not just appended text)
- Deep clone is verified: the original resume object is not mutated
- Validation uses the existing `validateResume` from `server/index.js` (import or duplicate the logic â€” prefer extracting to a shared util if needed, but do not break existing imports)

**action:**
- Create `server/lib/tailor/applyPatches.js`
- Export `applyPatches(resume, suggestions, decisions)` function
- Inside: `JSON.parse(JSON.stringify(resume))` for deep clone
- Filter suggestions to only those with `decisions[id]?.status === 'accepted' || decisions[id]?.status === 'edited'`
- For each accepted/edited suggestion, apply the patch based on `section` + `type`:
  - `summary` + `modify`: `cloned.summary = decisions[id].editedContent || suggestion.suggested`
  - `summary` + `add`: `cloned.summary = suggestion.suggested`
  - `skills` + `add`: `cloned.skills.push(suggestion.suggested)`
  - `experience` + `add`: find last entry in `cloned.experience`, push `suggestion.suggested` to its `bullets`; if empty, push `{ company: '', role: '', period: '', bullets: [suggestion.suggested] }`
  - `experience` + `modify`: find bullet matching `suggestion.current` in last experience entry, replace with `decisions[id].editedContent || suggestion.suggested`
  - `projects` + `add`: same pattern as experience but with `{ name: '', description: '', bullets: [...] }`
  - `projects` + `modify`: same pattern as experience modify
  - `remove` types: for skills, filter out the matching skill; for experience/projects bullets, splice the matching bullet; for summary, set to empty string
- After all patches applied, validate with `validateResume(cloned)` (inline the validation logic or import from a shared module)
- Return `{ resume: cloned, validation: validateResume(cloned) }`

### Task 2: Add draft CRUD routes and tailor endpoint to server

**read_first:**
- `server/index.js` (full file â€” understand existing route patterns, `generateId()`, `readJSON`/`writeJSON`, `readResumeVersion`, `readLibraryIndex`, `writeResumeVersion`, `writeLibraryIndex`, `validateResume`, `LIBRARY_DIR`, `VALID_ID`)
- `server/lib/tailor/applyPatches.js` (the module from Task 1)
- `resume_library/index.json` (index structure with `selected_id` and `versions` array)

**acceptance_criteria:**
- `POST /api/drafts` accepts `{ resume_id, posting_id, suggestions, decisions }` and returns `{ ok: true, draft: { id, resume_id, posting_id, company, role, suggestions, decisions, created_at } }`
- `GET /api/drafts/:id` returns the draft object with an added `tailored_resume` field containing the result of `applyPatches(source_resume, suggestions, decisions)`
- `POST /api/drafts/:id/save` accepts `{ name }`, saves the tailored resume as a new resume version via `writeResumeVersion`, adds the version to the library index with `source_id` pointing to the source resume, and deletes the draft file
- `DELETE /api/drafts/:id` deletes the draft file and returns `{ ok: true }`
- Drafts are stored as individual JSON files in `server/data/drafts/` (created on first draft write)
- `company` and `role` are resolved from the job posting (looked up via `posting_id` in `job_postings.json`)
- The `POST /api/drafts/:id/save` route validates the tailored resume with `validateResume` before saving; returns 400 if validation fails
- After save, the new resume version gets `source_id` set to the original `resume_id` from the draft
- Invalid draft IDs (failing `VALID_ID` test) return 400
- Missing drafts return 404

**action:**
- In `server/index.js`, add constants: `DRAFTS_DIR = path.join(DATA_DIR, 'drafts')`
- Add helper functions: `readDraft(id)` reads `DRAFTS_DIR/{id}.json`, `writeDraft(id, data)` writes to same, `deleteDraftFile(id)` removes the file. All check `VALID_ID` first.
- Add `ensureDraftsDir()` that creates the directory if missing (called before writes)
- Add `POST /api/drafts` route:
  - Extract `resume_id`, `posting_id`, `suggestions`, `decisions` from `req.body`
  - Validate `resume_id` and `posting_id` exist (lookup in library and job_postings.json)
  - Resolve `company` and `role` from the posting
  - Generate draft ID with `generateId()`
  - Write draft file with `{ id, resume_id, posting_id, company, role, suggestions, decisions, created_at: new Date().toISOString().split('T')[0] }`
  - Return `{ ok: true, draft }`
- Add `GET /api/drafts/:id` route:
  - Read draft, return 404 if missing
  - Read source resume via `readResumeVersion(draft.resume_id)`
  - Call `applyPatches(sourceResume, draft.suggestions, draft.decisions)`
  - Return `{ ...draft, tailored_resume: result.resume, validation: result.validation }`
- Add `POST /api/drafts/:id/save` route:
  - Read draft, return 404 if missing
  - Read source resume, apply patches
  - If `validation.ok` is false, return 400 with validation errors
  - Generate new resume version ID, write the tailored resume via `writeResumeVersion`
  - Add entry to library index: `{ id: newId, name: req.body.name || draft.company + ' - ' + draft.role, created_at: today, updated_at: today, source_id: draft.resume_id }`
  - Write library index
  - Delete draft file
  - Return `{ ok: true, version: newEntry }`
- Add `DELETE /api/drafts/:id` route:
  - Delete draft file, return 404 if missing, `{ ok: true }` on success
