# Plan 12-01: Backend -- Draft Storage API + Patch Application Engine (Revised)

## Objective

Build the server-side infrastructure for tailored resume generation: a patch application engine that applies accepted suggestions to a deep copy of the source resume, and ephemeral draft storage so accept/reject decisions survive navigation, browser refresh, and server restarts.

Purpose: This is the correctness boundary for Phase 12. Every suggestion must apply deterministically, every draft must persist across refreshes, and every saved resume must validate before entering the library.

Output: `server/lib/tailor/applyPatches.js`, `server/lib/validateResume.js`, four draft CRUD endpoints on the Express server.

    **Step B: Create applyPatches module**

    Create `server/lib/tailor/applyPatches.js` exporting `applyPatches(resume, suggestions, decisions)`:

    1. Deep clone: `JSON.parse(JSON.stringify(resume))` -- never mutate original
    2. Filter suggestions to only `decisions[id]?.status === 'accepted' || decisions[id]?.status === 'edited'`
    3. For each accepted/edited suggestion, apply by section+type:
       - `summary` + `modify`: replace `cloned.summary` with `decisions[id].editedContent || suggestion.suggested`
       - `summary` + `add`: set `cloned.summary = suggestion.suggested`
       - `skills` + `add`: push `suggestion.suggested` to `cloned.skills` (skip if already present -- dedup)
       - `skills` + `remove`: filter out matching skill string
       - `experience` + `add`: find last entry in `cloned.experience`, push `suggestion.suggested` to its `bullets`; if experience is empty, create `{ company: '', role: '', period: '', bullets: [suggestion.suggested] }`
       - `experience` + `modify`: **search ALL experience entries' bullets arrays** for a bullet containing `suggestion.current`, replace with `decisions[id].editedContent || suggestion.suggested`. If no match found, log warning and skip. Addresses review concern: modify patches only search last entry (Codex, OpenCode, Cursor consensus).
       - `experience` + `remove`: search all entries for bullet containing `suggestion.current`, splice it out
       - `projects` + `add`: same as experience but with `{ name: '', description: '', bullets: [...] }`
       - `projects` + `modify`: **search ALL project entries** for matching bullet, replace (same pattern as experience)
       - `projects` + `remove`: same search-all-entries pattern
    4. After all patches applied, validate with `validateResume(cloned)`
    5. Return `{ resume: cloned, validation: validateResume(cloned) }`
  </action>
