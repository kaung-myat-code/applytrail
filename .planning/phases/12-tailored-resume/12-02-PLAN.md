# Plan 12-02: Frontend -- Generate Button Wiring + Preview Page + Routing (Revised)

## Objective

Enable the "Generate Tailored Resume" button on the review page, persist accept/reject decisions via server-side drafts with URL-based identification, and create a dedicated preview page where the user can review the full tailored resume, edit its name, and save it to the library.

Purpose: This plan wires the frontend to the backend draft API, fixing the critical TAILOR-05 state restoration gap identified by all three reviewers. The key architectural decision is using URL search params (`?draft=<id>`) instead of React Router location state, so draft identification survives browser refresh.

Output: `client/src/pages/PreviewTailored.jsx`, `client/src/pages/PreviewTailored.module.css`, modified `ReviewSuggestions.jsx`, modified `main.jsx`.
