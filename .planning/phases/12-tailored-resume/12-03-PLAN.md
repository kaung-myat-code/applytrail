# Plan 12-03: Gap Closure -- Resume Editor Version-Aware Routing (G-12-2)

<objective>
Fix the UAT-reported gap where editing ANY resume-library card (including a freshly tailored/saved version) opens the Resume editor showing the original default resume instead of that card's actual content.

Root cause (confirmed in 12-UAT.md): the backend is correct -- `POST /api/drafts/:id/save` and `GET/PUT /api/resume-library/:id` already read/write the correct id-specific file. The bug is entirely frontend: `ResumeLibrary.jsx`'s Edit link always points to the static `/resume` route with no version id, and `Resume.jsx` has no id-aware fetch/save path, so it unconditionally calls the legacy singular `GET/PUT /api/resume` regardless of which card was clicked.

Purpose: Close gap G-12-2 so the tailored-resume workflow (Phase 12's core value) is actually usable end-to-end -- a user must be able to open and verify the tailored version they just generated.

Output: `/resume/:id` route, version-aware `Edit` links, and an id-aware `Resume.jsx` that falls back to the legacy endpoint only when no id is present.
</objective>
