Fix the confirmed "Invalid resume data" bug on Resume Library creation: `POST /api/resume-library` currently defaults `contact` to `{}` when no `resume_data` is supplied, which fails `validateResume`'s per-field contact checks. This is the single highest-priority confirmed bug from the phase's source UAT. Task 1 implements the D-08 default-value fix; Task 2 implements the D-09 regression-test requirement, kept intentionally narrow per D-10 (targeted regression coverage for this phase's fixes only, not a broader API contract suite).
