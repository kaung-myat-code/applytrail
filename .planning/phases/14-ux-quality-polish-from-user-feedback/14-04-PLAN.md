<objective>
Clarify the job-posting → application workflow: redirect from New Application straight to Cover Letter with a transitional confirmation on save, add an explicit "Save Application" confirmation step on Cover Letter that redirects to the Applications list, and clarify the "applied on" vs "last status change" date labels on the Applications list. Closes GitHub #2.

Purpose: Users currently save a job posting and land on a cleared form with no clear next step, and saving an application from Cover Letter leaves a dead-end "View in Applications" link instead of confirming and moving the user forward. Task 1 implements D-01, Task 2 implements D-02, and Task 3 implements D-04. Per D-03, this plan deliberately scopes the save-feedback fix to New Application and Cover Letter only — it does not introduce a shared toast/banner component reused by the Applications list or status-update flows, since that would be scope creep beyond the tracked GitHub issues.
Output: `NewApplication.jsx` redirects on save; `CoverLetter.jsx` gains an inline save-confirmation step that navigates to `/applications`; `Applications.jsx` date labels are unambiguous.
</objective>
