<scope_note>
Server-only fix. `client/src/pages/Analysis.jsx` reads the native fetch `res.ok` (the HTTP Response object, line 237) and then `data.report` / `data.suggestions` — it never inspects the echoed `validation` payload's `.ok`/`.valid` field. So the client is already consistent and must NOT be changed. Do not alter `validate.js`'s return shape either — `keywords.test.js`, `heuristic.test.js`, and the `validation` payload contract all already expect `{ valid, errors }`. The minimal, correct fix is to make `index.js` read `.valid`.
</scope_note>
