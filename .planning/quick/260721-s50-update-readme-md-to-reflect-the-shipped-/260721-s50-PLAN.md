## Verified v2.0 facts (source of truth — do not deviate)

Real page titles and flows:
- **Resume Library** (`/resume-library`, ResumeLibrary.jsx): create ("New Resume"), select, rename, delete, edit, Export PDF, Export JSON, and "Create Application" per version. Description in-app: "Manage multiple resume versions and select which one to use for analysis and tailoring."
- **Match Analysis** (`/analysis`, Analysis.jsx): pick a resume version + job posting + analysis provider, then "Analyze Match". Report shows a score out of 100, a summary, "Strengths", "Gaps to Address", a "Keyword Analysis" (Matched / Missing / Bonus), and a "Section Analysis" with per-section match-rate bars for Summary, Skills, Experience, Projects, Education. Leads to "Continue to Review Suggestions".
- **Review Suggestions** (`/analysis/review`, ReviewSuggestions.jsx): suggestions grouped by section; each can be Accepted, Rejected, or Edited; "Accept All" / "Reject All" bulk actions; a running "N of M accepted" count; "Generate Tailored Resume" (enabled once at least one is accepted). Side-by-side current-vs-suggested comparison is rendered by SuggestionCard.
- **Preview Tailored Resume** (`/analysis/preview`, PreviewTailored.jsx): shows the tailored resume, an editable name auto-filled as "Company - Role", a "Based on: <source name>" line, schema validation, "Back to Suggestions" (decisions preserved via server-side draft), and "Save to Library". After saving, a modal pre-fills a new application from the analyzed posting with the new resume version linked.

Export scope (CRITICAL): only **PDF** and **JSON** are implemented (`GET /api/resume-library/:id/export/pdf`, `GET /api/resume-library/:id/export/json`). PDF is generated with pdfmake. There is NO DOCX export endpoint — the earlier target-feature list mentioned DOCX but it was not shipped. Do not claim any export format other than PDF and JSON.

Screenshots that exist (do not reference any others): `docs/screenshots/dashboard.png`, `docs/screenshots/resume-editor.png`, `docs/screenshots/applications.png`. There are NO v2.0 page screenshots — do not add image tags for them.

Full v2.0 API route set (server/index.js), for the collapsible "API routes" table:
- `GET /api/resume-library`, `POST /api/resume-library`
- `GET /api/resume-library/:id`, `PUT /api/resume-library/:id`, `DELETE /api/resume-library/:id`
- `PUT /api/resume-library/:id/select`
- `GET /api/resume-library/:id/export/pdf`, `GET /api/resume-library/:id/export/json`
- `POST /api/analyze`
- `POST /api/drafts`, `GET /api/drafts/:id`, `POST /api/drafts/:id/save`, `DELETE /api/drafts/:id`
- `POST /api/generate-cover-letter`
- `PUT /api/applications/:id` (status update)

Application statuses (VALID_STATUSES): drafted, applied, interviewing, offered, rejected, withdrawn.
Analysis providers: heuristic (default, offline) plus Gemini, OpenRouter, Groq (with automatic fallback) — already covered by the existing "AI analysis providers" section.
