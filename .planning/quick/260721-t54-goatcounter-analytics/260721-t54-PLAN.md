<objective>
Add privacy-friendly GoatCounter pageview analytics to ApplyTrail, configurable via a Vite env var (VITE_GOATCOUNTER_SITE) so it is enabled in production (https://applytrail.onrender.com) and disabled by default on local dev.

Purpose: Give the deployed portfolio demo a lightweight, privacy-respecting pageview counter without a backend, without tracking during local development, and without hardcoding the site code.

Output: A runtime script-injector helper (analytics.js) wired into main.jsx, a unit test, and documentation of the new env var in client/.env.example and README.md.

<design_notes>
- Approach chosen: runtime script injection from a small helper module (idiomatic + unit-testable), NOT an inline script in index.html. Rationale: index.html is static and cannot read import.meta.env; a helper called from main.jsx keeps the env-gate logic in one testable place.
- VITE_GOATCOUNTER_SITE holds the GoatCounter site code (e.g. `applytrail`). The helper constructs the endpoint `https://<code>.goatcounter.com/count` and loads the async script `//gc.zgo.at/count.js`.
- Vite loads env files from the client/ directory (project root of the Vite app), so the example file lives at client/.env.example. On Render, the var is set in the service's environment (or render.yaml) at build time.
- Scope is a simple pageview counter (count on load via GoatCounter's standard snippet). SPA route-change counting is out of scope for this quick task.
</design_notes>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| app → third-party CDN (gc.zgo.at) | The browser loads and executes a remote third-party script (count.js) into the app's origin. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-t54-01 | Tampering | Remote count.js from gc.zgo.at executing in app origin | low | accept | GoatCounter's official CDN is a well-established privacy-focused analytics provider; the script is loaded async and only sends pageview beacons. Acceptable for a single-user portfolio demo. No user secrets exist in the client. |
| T-t54-02 | Information disclosure | Pageview data sent to GoatCounter | low | accept | GoatCounter is privacy-friendly (no cookies, no cross-site tracking). Only aggregate pageviews are collected. Disabled by default on local dev (env-gated), so developer browsing is not tracked. |
| T-t54-03 | Tampering | site code value injected into script attribute | low | mitigate | Site code originates only from a build-time env var controlled by the deployer (not user input), and is interpolated into a fixed goatcounter.com URL template — no untrusted-input path into the DOM. |
</threat_model>
