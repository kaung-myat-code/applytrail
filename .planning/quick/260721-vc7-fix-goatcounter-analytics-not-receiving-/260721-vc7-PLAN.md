<objective>
Fix GoatCounter analytics not receiving data in production by allowlisting the
GoatCounter host (gc.zgo.at) in the production helmet Content-Security-Policy.

Purpose: The production CSP in server/index.js currently sets `scriptSrc: ["'self'"]`
and `connectSrc: ["'self'"]`, which blocks both loading `//gc.zgo.at/count.js` and
the tracking beacon it sends. This was confirmed via live Chrome DevTools inspection
of https://applytrail.onrender.com (script request blocked, `fetch` to gc.zgo.at threw
"Failed to fetch"). The client-side wiring (client/src/lib/analytics.js, main.jsx) and
render.yaml VITE_GOATCOUNTER_SITE are already correct — only the CSP needs the host added.

Output: Updated helmet CSP in server/index.js with `https://gc.zgo.at` in both the
`scriptSrc` and `connectSrc` directives.
</objective>
